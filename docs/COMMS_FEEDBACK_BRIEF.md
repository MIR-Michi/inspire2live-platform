# Communications Team Feedback Brief

## Purpose
Use this 10-minute conversation to understand:
- whether the communications workspace would support the team's real work once released,
- what feels useful already,
- what feels unclear or missing before release,
- what should be added next.

## Suggested Opening
This is a pre-release feedback session. You are not expected to already work with the platform. We want to understand whether it looks useful for your communications work, what feels unclear, and what would need to be in place before release for you to actually use it.

## 10-Minute Flow
### 1. Context check (1 minute)
- Which part of your communications work matters most right now?
- What are the main tools or workarounds you currently rely on?

### 2. Fit for purpose (3 minutes)
- Based on what you have seen, does this platform look like it could support your communications workflow?
- Which part looks most useful?
- Which part feels confusing, unnecessary, or incomplete?

### 3. Friction and gaps (3 minutes)
- If you had to use this tomorrow, what would get in the way?
- Where would you still need to work outside the platform?
- What is missing that would make this usable for you?

### 4. Priority additions (2 minutes)
- If we could improve only one thing next, what should it be?
- What would need to be true before you would trust and use it?
- What should be added on top of the current setup?

### 5. Close (1 minute)
- If this were released soon, would you want to use it for your communications work?
- What is the main reason for your answer?

## Optional Follow-Up Questions
- Which information should always be visible at a glance?
- What should be automated?
- What should be simpler?
- What should never be inside this platform?
